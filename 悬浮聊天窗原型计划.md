# 悬浮聊天窗原型计划

## 目标

基于 `D:\悬浮聊天APP\zhifaAndroid` 现有 Android 无障碍边缘手势项目，先完成一个静态 Compose 悬浮聊天窗原型。不接真实聊天后端，不做真实消息发送，只验证核心能力：

- 通过 `AccessibilityService` 创建系统级悬浮窗口。
- 悬浮窗口内容使用 Jetpack Compose 绘制。
- 悬浮窗呈半透明聊天界面，可覆盖在其他 App 上方。
- 保留现有左右边缘手势能力，后续可扩展为控制悬浮聊天窗的自定义动作。

## 当前基础

项目已经具备：

- `UbikiAccessibilityService`：无障碍服务入口。
- `TYPE_ACCESSIBILITY_OVERLAY`：已用于左右边缘透明触控条。
- `EdgeOverlayView`：现有边缘手势触控层。
- `BackWaveOverlayController`：已证明可在无障碍悬浮层中挂载 `ComposeView`。
- `GestureType` / `GestureAction` / `UbikiPreferences`：已有手势动作映射模型。

## 第一阶段范围

本阶段只做静态原型：

- 新增一个 `FloatingChatOverlayController`。
- 在无障碍服务启动时显示一个半透明聊天悬浮窗。
- 聊天窗中展示固定假数据：
  - 顶部会话栏。
  - 左侧联系人头像列表。
  - 中间聊天消息气泡。
  - 右侧工具按钮栏。
  - 底部输入区域。
- 悬浮窗暂不接真实输入发送，不接网络，不接数据库。
- 暂不实现拖动、缩放、最小化和真实会话切换。

## 技术方案

- 复用 `ubiki-accessibility` 模块，因为悬浮窗生命周期依赖无障碍服务。
- 使用 `WindowManager.addView()` 添加 `ComposeView`。
- 窗口类型使用 `WindowManager.LayoutParams.TYPE_ACCESSIBILITY_OVERLAY`。
- 初版窗口尺寸占屏幕大部分区域，居中显示，背景半透明。
- 初版使用 `FLAG_NOT_FOCUSABLE`，优先保证不抢系统输入焦点；输入框先作为视觉占位。
- Compose UI 拆成纯函数，静态数据模型放在 Kotlin data class 中，方便后续测试和替换真实数据。

## 后续演进

第二阶段可以继续做：

- 新增手势动作：显示/隐藏聊天窗、展开/收起、切换联系人。
- 将边缘手势映射到悬浮聊天窗控制动作。
- 增加悬浮窗拖动、贴边和最小化气泡。
- 处理输入框焦点和软键盘。
- 接入真实消息数据层。

## 验收标准

- `:app:assembleDebug` 构建成功。
- 安装到手机后，开启无障碍服务时能看到半透明聊天悬浮窗。
- 现有边缘触控条仍可工作。
- 悬浮窗 UI 使用 Compose 实现。
- 静态 UI 在手机屏幕上不崩溃、不黑屏、不遮挡到无法关闭。
